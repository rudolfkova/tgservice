# Telegram Service (Go + gotd + gRPC)

## Описание

Сервис на Go, поддерживающий несколько независимых соединений с Telegram через библиотеку `github.com/gotd/td`.

Сервис предоставляет gRPC API для:

* динамического создания и удаления Telegram-сессий
* отправки текстовых сообщений
* получения входящих текстовых сообщений (stream)

Сессии полностью изолированы друг от друга. Ошибки или завершение одной сессии не влияют на работу остальных.

---

## Жизненный цикл сессии

### Создание сессии

1. Клиент вызывает `CreateSession`
2. Сервис:
   * создаёт Telegram client
   * инициирует QR-авторизацию
   * возвращает строку для генерации QR-кода
3. Пользователь сканирует QR в Telegram:
   * Settings - Devices - Scan QR
4. После подтверждения сессия становится активной

---

### Отправка сообщения

Метод: `SendMessage`

* Указывается `session_id`
* Указывается `peer` (например `@username`)
* Сообщения отправляются конкурентно
* Возвращается `message_id`

---

### Получение сообщений

Метод: `SubscribeMessages`

* gRPC streaming
* Возвращает входящие текстовые сообщения:

  * ID сообщения
  * отправитель
  * текст
  * timestamp

Каждая подписка изолирована в рамках своей сессии.

---

### Удаление сессии

Метод: `DeleteSession`

* Telegram клиент останавливается
* Освобождаются ресурсы
* При необходимости вызывается `auth.logOut`
* Сессия удаляется из памяти

---

## Конкурентность и изоляция

* Каждая Telegram-сессия работает независимо
* Доступ к хранилищу сессий синхронизирован
* Параллельная отправка сообщений поддерживается
* Потоки подписки не блокируют друг друга
* Контексты используются для корректного завершения работы

Проблема в одной сессии не приводит к остановке сервиса.

Конфигурация задаётся через переменные окружения:

```
TELEGRAM_API_ID=
TELEGRAM_API_HASH=
GRPC_PORT=
```

## Сборка и запуск

### 1. Установка зависимостей

```bash
go mod tidy
```

### 2. Генерация protobuf

```bash
protoc --go_out=. --go-grpc_out=. proto/telegram.proto
```

### 3. Запуск

```bash
go run ./cmd/app
```

---

## Пример вызова через grpcurl

### Создание сессии

```bash
grpcurl -plaintext \
  -d '{}' \
  localhost:50051 pact.telegram.TelegramService/CreateSession
```

Ответ:

```json
{
  "session_id": "uuid",
  "qr_code": "tg://login?token=..."
}
```

---

### Отправка сообщения

```bash
grpcurl -plaintext \
  -d '{"session_id":"uuid","peer":"@username","text":"Hello"}' \
  localhost:50051 pact.telegram.TelegramService/SendMessage
```

---

### Подписка на сообщения

```bash
grpcurl -plaintext \
  -d '{"session_id":"uuid"}' \
  localhost:50051 pact.telegram.TelegramService/SubscribeMessages
```

## Тестирование

Покрыты тестами:

* usecase слой
* gRPC handler
* бизнес-логика работы с сессиями
* отправка сообщений

Сам драйвер не покрыт тестами, т.к. для тестирования Telegram API нужно реальное подключение.

Используются моки для изоляции зависимостей.

Запуск тестов:

```bash
go test ./...
```
